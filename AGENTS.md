## Strict Constraints

**Naming — zero abbreviations.**
No single-letter variables (`i`, `e`) or shorthand acronyms (`ct`, `pts`). Use full semantic names (`creditTransaction`). Never use generic names like `parsed`, `data`, or `result` — use specific names like `parsedCampaigns`. Never use `raw` as a prefix; a name without a prefix is the non-adapted value. Only use `adapted` prefix for transformed objects (`adaptedStores`).

**Syntax — prefer destructuring.**
Always destructure from objects and arrays instead of dot notation or index access (`const { currency } = region` not `const currency = region.currency`).

**Comments — zero.**
Code must be self-explanatory.

**TypeScript — no `any` or `unknown`.**
Explicit return types only on: exported functions, custom hooks, type/interface definitions. Internal functions rely on inference.

**Returns.**
- One-liner guards if `if` + return fits under 80 chars: `if (!data) return ""`
- No direct invocation returns — assign to a semantic variable first, then return
- No ternary returns — use explicit `if` blocks and early returns
- No implicit arrow returns — always use `{ return ... }`
- Early returns over `if/else` cascades

**Promises.** Never return raw Promises. Always `await` and return the resolved value.

**React imports.** Never use the global `React` object. Import types/hooks directly.

**Named handlers.** No inline function props. Extract as named declarations and pass by reference. Inline arrow functions in JSX create new references every render.

**Readability.** Group code by meaning. Use empty lines to separate logical steps (declarations, logic, return).

**Git commits.** No AI attribution. No "Co-authored-by", "Generated by", or any mention of the AI model. Commits must look like they were written by the user.

## Architecture & Best Practices

- Follow `vercel-react-best-practices` and `frontend-design` (A11y)
- Mirror local project style — match existing conventions for filenames and declarations
- Always use **Vitest** + **React Testing Library** for unit tests

## Post-Generation Checklist

- [ ] All variables are full semantic words (no `ct`, `item`, `i`)
- [ ] Code is grouped into logical blocks with empty lines
- [ ] Hooks/types imported directly (no `React.useState`)
- [ ] Style matches existing project files
- [ ] Semantic HTML and ARIA attributes included
- [ ] `if` blocks used instead of ternary returns
- [ ] Variables named by meaning (`parsedCampaigns`), `adapted` prefix only for transformed objects
- [ ] No cascaded ternaries
- [ ] Linter passes (`npm run lint` or equivalent)
- [ ] Unit and E2E tests green

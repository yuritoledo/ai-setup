## Strict Constraints

**Naming — zero abbreviations.**
No single-letter variables (`i`, `e`) or shorthand acronyms (`ct`, `pts`). Use full semantic names (`creditTransaction`). Never use generic names like `parsed`, `data`, or `result` — use specific names like `parsedCampaigns`. Never use `raw` as a prefix; a name without a prefix is the non-adapted value. Only use `adapted` prefix for transformed objects (`adaptedStores`).

**Syntax — prefer destructuring.**
Always destructure from objects and arrays instead of dot notation or index access (`const { currency } = region` not `const currency = region.currency`).

**Comments — zero.**
Code must be self-explanatory.

**TypeScript — no `any` or `unknown`.**
Explicit return types only on: exported functions, custom hooks, type/interface definitions. Internal functions rely on inference.

**Props types.** Use `type Props = { ... }`, never `interface` for component props.

**Returns.**
- One-liner guards if `if` + return fits under 80 chars: `if (!data) return ""`
- No direct invocation returns — assign to a semantic variable first, then return
- No ternary returns — use explicit `if` blocks and early returns
- No implicit arrow returns — always use `{ return ... }`
- Early returns over `if/else` cascades

**Promises.** Never return raw Promises. Always `await` and return the resolved value.

**React imports.** Never use the global `React` object. Import types/hooks directly.

**Named handlers.** No inline function props. Extract as named declarations and pass by reference. Inline arrow functions in JSX create new references every render. Prefer `onX` over `handleX` for event handler names.

**Function declarations.** Use `function` declarations, not arrow functions, except inside `.map()`, `.filter()`, `.reduce()` callbacks.

**Hook return values.** Assign hook result to a variable; access properties on it. Do not rename via destructuring.
```ts
const tournamentsQuery = useTournamentsQuery()  // ✅
const { data } = useTournamentsQuery()          // ❌ — renaming loses context
```

**Conditional classes.** Use `cn("base", condition && "modifier")` — never `` `base${cond ? " modifier" : ""}` ``. Prettier strips the leading space from template literals, silently producing invalid class names. For state-driven styles prefer `data-*` attributes with Tailwind's `data-[key=value]:` variant over conditional class merging.

**No ternary waterfall.** Never use chained ternaries for loading/error/empty/data states. Extract a separate component with early returns.

**No IIFE in JSX.** Extract logic before the `return` — never use immediately-invoked functions inside JSX.

**One component per file.** Extract sub-components into their own files.

**`"use client"` placement.** Only in `page.tsx` files. All other files omit it.

**Readability.** Group code by meaning. Use empty lines to separate logical steps (declarations, logic, return).

**Git commits.** No AI attribution. No "Co-authored-by", "Generated by", or any mention of the AI model. Commits must look like they were written by the user.

## Agent Dispatching

- **Parallel first.** Break work into independent subtasks and run them concurrently whenever there is no shared mutable state.
- **Serialize only when required.** Sequential execution only for dependent steps, shared-state mutations, or conflicting operations.
- Default: dispatch as many agents as possible in parallel for independent or read-only tasks.

## E2E Conventions

**Selector preference: `getByTestId` over role queries.**
Never query buttons by role and label. Add `data-testid` to source and use `getByTestId` / `queryByTestId`.

```ts
// ❌
await page.getByRole("button", { name: "Delete" }).click()
// ✅
await page.getByTestId("delete-button").click()
```

**Cookie injection (preferred over sidebar interaction).**
Both Jotai atoms read from cookies on init. Inject before `page.goto()` to skip sidebar combobox clicks:

```ts
await page.context().addCookies([
  { name: "tenantId", value: "2", url: "http://localhost:3000" },
  { name: "appId", value: "83", url: "http://localhost:3000" },
])
await page.goto("/target-page")
```

Rules: inject before any `goto()`, scope to `http://localhost:3000`, skip `page.goto("/")` + sidebar interaction entirely.

**Table data wait.** Replace `waitForLoadState("networkidle")` with skeleton assertion:
```ts
await expect(page.getByTestId("skeleton-row")).toHaveCount(0)
```

**Sidebar loading wait (when cookie injection not possible):**
```ts
await page.goto("/")
await expect(page.locator("nav[data-hidden=false]")).toBeVisible({ timeout: 10000 })
```

**Gotchas.**
- **React Query caching breaks `waitForRequest()`** — when filter returns to original value, RQ serves cache without new request. Use `page.waitForTimeout(500)` + UI assertion for the "return to original" step.
- **Re-render race on filter chips** — clicking one chip can remove another before you click it. Fix: use `page.evaluate()` to click both in the same synchronous microtask.
- **Graceful skip is a feature** — `test.skip(true, "No X data in test environment")` distinguishes missing data from broken feature. Never hard-fail on missing test data.

**Use the `e2e-table-filter` skill** when validating that a filter actually changes table row data (not just that API fires).
